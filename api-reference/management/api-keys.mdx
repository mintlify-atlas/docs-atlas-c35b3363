title: API Keys

API keys allow your applications to access the LLM Gateway. They can be scoped to specific projects and have usage limits and IAM rules applied.

## Create API Key

<Note>
API keys are prefixed with `llmgtwy_` in production and `llmgdev_` in development.
</Note>

<ParamField path="description" type="string" required>
  Human-readable description of the API key (1-255 characters)
</ParamField>

<ParamField path="projectId" type="string" required>
  ID of the project this key belongs to
</ParamField>

<ParamField path="usageLimit" type="string | null">
  Maximum spend limit for this key in USD (e.g., "100.00"). Null for unlimited.
</ParamField>

<ResponseField name="apiKey" type="object">
  The created API key object with the full token (only shown once)
  
  <Expandable title="API Key Object">
    <ResponseField name="id" type="string">
      Unique identifier
    </ResponseField>
    
    <ResponseField name="token" type="string">
      The full API key token (only returned on creation)
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Description of the key
    </ResponseField>
    
    <ResponseField name="status" type="'active' | 'inactive' | 'deleted'">
      Current status of the key
    </ResponseField>
    
    <ResponseField name="usageLimit" type="string | null">
      Maximum spend limit in USD
    </ResponseField>
    
    <ResponseField name="usage" type="string">
      Current usage/spend in USD
    </ResponseField>
    
    <ResponseField name="projectId" type="string">
      Associated project ID
    </ResponseField>
    
    <ResponseField name="createdBy" type="string">
      User ID who created the key
    </ResponseField>
    
    <ResponseField name="createdAt" type="date">
      Creation timestamp
    </ResponseField>
    
    <ResponseField name="updatedAt" type="date">
      Last update timestamp
    </ResponseField>
    
    <ResponseField name="creator" type="object | null">
      Information about the user who created the key
      
      <Expandable>
        <ResponseField name="id" type="string">User ID</ResponseField>
        <ResponseField name="name" type="string | null">User name</ResponseField>
        <ResponseField name="email" type="string">User email</ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="iamRules" type="array">
      IAM rules attached to this key
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.llmgateway.io/keys/api \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Production API Key",
    "projectId": "proj_abc123",
    "usageLimit": "100.00"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.llmgateway.io/keys/api', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_SESSION_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    description: 'Production API Key',
    projectId: 'proj_abc123',
    usageLimit: '100.00'
  })
});

const data = await response.json();
console.log('API Key:', data.apiKey.token);
```

```python Python
import requests

response = requests.post(
    'https://api.llmgateway.io/keys/api',
    headers={'Authorization': 'Bearer YOUR_SESSION_TOKEN'},
    json={
        'description': 'Production API Key',
        'projectId': 'proj_abc123',
        'usageLimit': '100.00'
    }
)

api_key = response.json()['apiKey']['token']
print(f'API Key: {api_key}')
```
</RequestExample>

<ResponseExample>
```json 200 Response
{
  "apiKey": {
    "id": "key_xyz789",
    "token": "llmgtwy_abcdef1234567890",
    "description": "Production API Key",
```

The full API key token is only returned once during creation. Store it securely - you won't be able to retrieve it again.
</Warning>

## List API Keys

Retrieve all API keys you have access to.

<ParamField query="projectId" type="string">
  Filter keys by project ID
</ParamField>

<ParamField query="filter" type="'mine' | 'all'">
  Filter by creator:
  - `mine`: Only your API keys
  - `all`: All keys in accessible projects (requires admin/owner role)
</ParamField>

<ResponseField name="apiKeys" type="array">
  Array of API key objects (with masked tokens)
</ResponseField>

<ResponseField name="planLimits" type="object | undefined">
  Plan limits information (only when projectId is specified)
  
  <Expandable>
    <ResponseField name="currentCount" type="number">Number of active keys</ResponseField>
    <ResponseField name="maxKeys" type="number">Maximum allowed keys</ResponseField>
    <ResponseField name="plan" type="'free' | 'pro'">Organization plan</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="userRole" type="'owner' | 'admin' | 'developer'">
  Your role in the organization
</ResponseField>

<RequestExample>
```bash cURL
curl https://api.llmgateway.io/keys/api?projectId=proj_abc123&filter=mine \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```
</RequestExample>

## Update API Key Status

Activate or deactivate an API key.

<ParamField path="id" type="string" required>
  API key ID
</ParamField>

<ParamField body="status" type="'active' | 'inactive'" required>
  New status for the API key
</ParamField>

<RequestExample>
```bash cURL
curl -X PATCH https://api.llmgateway.io/keys/api/key_xyz789 \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "inactive"}'
```
</RequestExample>

<ResponseExample>

```json 200 Response
    "createdBy": "user_123",
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-16T14:20:00Z"
  }
}
```
</ResponseExample>

## Update Usage Limit

Modify the spending limit for an API key.

<ParamField path="id" type="string" required>
  API key ID
</ParamField>

<ParamField body="usageLimit" type="string | null" required>
  New usage limit in USD, or null for unlimited
</ParamField>

<RequestExample>
```bash cURL
curl -X PATCH https://api.llmgateway.io/keys/api/limit/key_xyz789 \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"usageLimit": "200.00"}'
```
</RequestExample>

## Delete API Key

Soft-delete an API key. The key will be marked as deleted and can no longer be used.

<ParamField path="id" type="string" required>
  API key ID
</ParamField>

<Note>
Developers can only delete their own API keys. Admins and owners can delete any key in their organization.
</Note>

<RequestExample>
```bash cURL
curl -X DELETE https://api.llmgateway.io/keys/api/key_xyz789 \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```
</RequestExample>

<ResponseExample>
```json 200 Response
{
  "message": "API key deleted successfully"
}
```
</ResponseExample>

## IAM Rules

IAM rules allow fine-grained access control for API keys.

### Rule Types

- `allow_models`: Whitelist specific models
- `deny_models`: Blacklist specific models
- `allow_pricing`: Restrict to free or paid models
- `deny_pricing`: Exclude free or paid models
- `allow_providers`: Whitelist specific providers
- `deny_providers`: Blacklist specific providers

### Create IAM Rule

<ParamField path="id" type="string" required>
  API key ID
</ParamField>

<ParamField body="ruleType" type="string" required>
  Type of IAM rule (see Rule Types above)
</ParamField>

<ParamField body="ruleValue" type="object" required>
  Rule configuration based on type
  
  <Expandable>
    <ParamField body="models" type="string[]">
      Array of model IDs (for model rules)
    </ParamField>
    <ParamField body="providers" type="string[]">
      Array of provider IDs (for provider rules)
    </ParamField>
    <ParamField body="pricingType" type="'free' | 'paid'">
      Pricing type (for pricing rules)
    </ParamField>
    <ParamField body="maxInputPrice" type="number">
      Max input token price
    </ParamField>
    <ParamField body="maxOutputPrice" type="number">
      Max output token price
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="status" type="'active' | 'inactive'" default="active">
  Initial status of the rule
</ParamField>

<RequestExample>
```bash cURL
curl -X POST https://api.llmgateway.io/keys/api/key_xyz789/iam \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ruleType": "allow_models",
    "ruleValue": {
      "models": ["gpt-4o", "claude-3-5-sonnet-20241022"]
    },
    "status": "active"
  }'
```
</RequestExample>

<ResponseExample>
```json 200 Response
{
  "message": "IAM rule created successfully",
  "rule": {
    "id": "rule_123",
    "apiKeyId": "key_xyz789",
    "ruleType": "allow_models",
    "ruleValue": {
      "models": ["gpt-4o", "claude-3-5-sonnet-20241022"]
    },
    "status": "active",
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-15T10:30:00Z"
  }
}
```
</ResponseExample>
