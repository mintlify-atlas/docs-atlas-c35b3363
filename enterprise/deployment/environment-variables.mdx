## Overview

LLM Gateway configuration is managed through environment variables. This page documents all available variables grouped by service.

## Database Configuration

### PostgreSQL

<ParamField path="POSTGRES_USER" type="string" default="postgres">
PostgreSQL username for authentication.
</ParamField>

<ParamField path="POSTGRES_PASSWORD" type="string" required>
PostgreSQL password. Use a strong password in production.
</ParamField>

<ParamField path="POSTGRES_DB" type="string" default="llmgateway">
PostgreSQL database name.
</ParamField>

<ParamField path="POSTGRES_PORT" type="number" default="5432">
PostgreSQL server port.
</ParamField>

<ParamField path="DATABASE_URL" type="string" required>
Complete PostgreSQL connection URL.

Format: `postgres://user:password@host:port/database`

Example: `postgres://postgres:pw@postgres:5432/llmgateway`
</ParamField>

## Authentication & Security

<ParamField path="AUTH_SECRET" type="string" required>
Secret key for session encryption. Generate with:
```bash
openssl rand -base64 32
```
</ParamField>

<ParamField path="COOKIE_DOMAIN" type="string" default="localhost">
Domain for authentication cookies.

Example: `yourdomain.com`
</ParamField>

<ParamField path="PASSKEY_RP_ID" type="string" default="localhost">
Relying Party ID for WebAuthn passkeys. Should match your domain.

Example: `yourdomain.com`
</ParamField>

<ParamField path="PASSKEY_RP_NAME" type="string" default="LLMGateway">
Relying Party display name for WebAuthn passkeys.
</ParamField>

## Timeout Configuration

<ParamField path="TIMEOUT_MS" type="number" default="5000">
Health check timeout in milliseconds.
</ParamField>

<ParamField path="KEEP_ALIVE_TIMEOUT_S" type="number" default="620">
HTTP server keep-alive timeout in seconds. Should be higher than your load balancer's keepalive timeout to prevent 502 errors.

Default is 620s for gateway and 60s for API. GCP Load Balancer uses 600s, so gateway default is set higher.
</ParamField>

<ParamField path="GATEWAY_TIMEOUT_MS" type="number" default="300000">
Maximum time for end-to-end gateway request in milliseconds (5 minutes).
</ParamField>

<ParamField path="AI_STREAMING_TIMEOUT_MS" type="number" default="240000">
AI provider request timeout for streaming requests in milliseconds (4 minutes).

Should be shorter than `GATEWAY_TIMEOUT_MS` to allow for error handling. Defaults to 80% of `GATEWAY_TIMEOUT_MS`.
</ParamField>

<ParamField path="AI_TIMEOUT_MS" type="number" default="180000">
AI provider request timeout for non-streaming requests in milliseconds (3 minutes).

Non-streaming requests use a shorter timeout since they don't benefit from incremental responses.
</ParamField>

## Worker Configuration

<ParamField path="BACKFILL_DURATION_SECONDS" type="number" default="300">
Stats calculator backfill duration in seconds (5 minutes).
</ParamField>

<ParamField path="ENABLE_DATA_RETENTION_CLEANUP" type="boolean" default="false">
Enable automatic deletion of verbose log data after retention period.

Set to `"true"` to enable. When enabled, verbose fields are nullified after 30 days for all users.
</ParamField>

<ParamField path="EMAIL_FOLLOW_UPS" type="boolean" default="false">
Enable follow-up lifecycle emails.

Set to `"true"` to enable. Default just logs events without sending emails.
</ParamField>

## Feature Flags

<ParamField path="PLATFORM_FEE_PERCENTAGE" type="number" default="0.05">
Platform fee percentage for credit usage (5%).

Applied when users pay with credits using platform API keys.
</ParamField>

<ParamField path="BILL_CANCELLED_REQUESTS" type="boolean" default="true">
Bill cancelled requests based on estimated token usage.

When enabled, cancelled requests are billed for prompt tokens and any partial completion tokens that were streamed before cancellation, plus request costs.

Set to `"false"` to disable.
</ParamField>

<ParamField path="IMAGE_SIZE_LIMIT_MB" type="number" default="100">
Maximum image upload size in megabytes.
</ParamField>

## Example .env File

```bash
# Copy this to .env and update the values

# Database
POSTGRES_USER=postgres
POSTGRES_PASSWORD=change_this_secure_password
POSTGRES_DB=llmgateway
POSTGRES_PORT=5432
DATABASE_URL=postgres://postgres:change_this_secure_password@postgres:5432/llmgateway

# Redis
REDIS_PASSWORD=change_this_redis_password
REDIS_PORT=6379

# Service Ports
GATEWAY_PORT=4001
API_PORT=4002
UI_PORT=3002
PLAYGROUND_PORT=3003
ADMIN_PORT=3006

# URLs (update for your domain)
UI_URL=https://llmgateway.yourdomain.com
API_URL=https://api.llmgateway.yourdomain.com
ORIGIN_URLS=https://llmgateway.yourdomain.com

# Authentication
AUTH_SECRET=generate-with-openssl-rand-base64-32
COOKIE_DOMAIN=yourdomain.com
PASSKEY_RP_ID=yourdomain.com
PASSKEY_RP_NAME=LLMGateway

# GitHub OAuth (optional)
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

# Google OAuth (optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# LLM Providers (add the ones you need)
LLM_OPENAI_API_KEY=sk-your_openai_key_here
LLM_ANTHROPIC_API_KEY=sk-ant-your_anthropic_key_here

# Stripe (optional)
STRIPE_SECRET_KEY=sk_live_your-key
STRIPE_WEBHOOK_SECRET=whsec_your-secret
STRIPE_PRO_MONTHLY_PRICE_ID=price_your-id
STRIPE_PRO_YEARLY_PRICE_ID=price_your-id

# Email (optional)
RESEND_API_KEY=re_your-key

# Analytics (optional)
POSTHOG_KEY=your-key
POSTHOG_HOST=https://app.posthog.com
```

## Security Best Practices

<Warning>
Never commit `.env` files to version control. Add `.env` to your `.gitignore`.
</Warning>

### Generate Secure Secrets

```bash
# Generate AUTH_SECRET
openssl rand -base64 32

# Generate passwords
openssl rand -base64 24
```

### Use Secrets Management

For production:

- **Docker** - Use Docker secrets
- **Kubernetes** - Use Kubernetes secrets or external secrets operator
- **Cloud** - Use AWS Secrets Manager, Google Secret Manager, Azure Key Vault
- **HashiCorp Vault** - For on-premise deployments

### Rotate Credentials

Regularly rotate:

- Database passwords
- Redis passwords
- AUTH_SECRET
- API keys
- OAuth secrets
